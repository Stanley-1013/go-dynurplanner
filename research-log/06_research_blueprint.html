Loop 6 · Research Blueprint · Incorporating advisor's occupancy-grid directive

GO-DynURPlanner：柵格觀測 × 解析真值 ×連續時間安全的完整研究藍圖

師兄（URPlanner 一作）給了明確指令：障礙區域用 occupancy grid 表徵、作為狀態觀測的一部分、與 RL 結合而非退回 search-based。本輪以此為錨，對前五輪的全部積累做一次頂層整合與批判性審計——找出目前計畫的四個真實漏洞並逐一修補，其中最關鍵的修補（佔據預報輔助任務）同時成為整個系統的拱心石。產出：完整的系統設計、形式化核心（含一個可證明的保守性引理與具體數字）、以「穿隧率」為 Figure-1 的實驗設計、誠實的新穎性帳本、與分階段里程碑。

Critical Audit

批判審計：目前計畫（Loop 5 + 師兄指令）的四個真實漏洞

把自己當成 RA-L/CoRL 審稿人來打自己的計畫，找到四個非修不可的洞。每個都附修法——其中洞①的修法會升級成整篇論文的拱心石。

洞① · 觀測-獎勵資訊鴻溝（最嚴重，可致訓練直接失敗）

計畫讓 policy 觀測 occupancy grid、但獎勵（D-UOAR-CT）依賴障礙速度的特權資訊。問題：單幀佔據柵格不含任何速度資訊——同一個柵格快照，障礙「正在逼近」與「正在遠離」完全不可分，價值函數學不出來（POMDP 崩壞，Loop 2 已引 Hart et al. 2112.12465 證明幀堆疊/LSTM 都不能穩定完全恢復缺失的速度）。若只是天真地堆疊 k 幀丟給 3D-CNN，速度資訊「原則上在裡面」但網路未必抽得出來——這正是 Hart 實證的失效模式。修法見拱心石一節：用佔據預報（occupancy forecasting）輔助任務強制把運動資訊壓進潛向量。

洞② · τ-線性化誤差從未被定量（審稿人必問）

Part⑤ 的連續時間推導假設連桿端點在 [t,t+Δt] 內走直線（弦近似），但關節旋轉使真實軌跡是弧。Loop 3 誠實列了這個未決問題但沒解。修法：把它變成引理而非道歉——旋轉 δθ 的關節、半徑 R 處的點，弧對弦的最大偏差 = R(1−cos(δθ/2)) ≈ Rδθ²/8。具體數字：δθ=0.1 rad、R=1m → 單關節 1.25mm；7 關節同動保守疊加 ≈ 8.75mm。把這個界加進膨脹盒的安全偏移，連續時間檢查就從「近似」變成「可證明保守」（見保守性引理）。~1cm 的膨脹代價換一個定理，穩賺。

洞③ · 「柵格當狀態」本身不是新穎性（先行文獻存在）

本輪查證摘要層級：voxel/occupancy 特徵當 manipulator RL 狀態已有先例（DexRepNet 抓取、離散規劃橋接工作等），移動機器人領域也已有「速度估計輔助任務」（MDRLAT, Sensors 2021）。所以論文不能把「用了柵格」當貢獻宣稱。修法：新穎性錨定在三個真正沒人做的組合點——(i) 佔據預報（而非速度回歸）作為 manipulator 動態避障的輔助任務；(ii) 柵格觀測與解析連續時間碰撞真值的師徒式耦合；(iii) 柵格由參數化空間解析光柵化生成、全程免模擬器（下詳）。

洞④ · 屏蔽（Shield）的部署斷層

訓練時屏蔽用特權 AABB+速度做區間根檢查——但部署時真實世界只有深度相機的體素/點雲，沒有乾淨的盒參數與追蹤好的速度，屏蔽豈不失效？修法（本輪新發現，很優雅）：體素本身就是小 AABB——同一套 λ(τ) 區間根數學逐體素適用，零改動；體素的「速度」不需要顯式追蹤，直接用拱心石的佔據預報器輸出的未來佔據做檢查對象。屏蔽從「物件級盒」平滑退化到「體素級」，部署閉環完成。廣義相位剪枝（只查臂附近體素）控制成本。

The Keystone

拱心石：佔據預報輔助任務（Occupancy Forecasting as Auxiliary Task）

一句話

讓柵格編碼器在輸出策略潛向量 zt 的同時，被迫預測 t+Δt 的佔據柵格（自監督輔助頭）。預測未來佔據的唯一途徑是從歷史幀中抽出運動資訊——所以這個輔助損失以結構強制的方式把速度/趨勢壓進 zt，直接治癒洞①的 POMDP 崩壞，不需要任何人工設計的速度輸入或顯式追蹤。

為什麼這一手在 URPlanner 生態裡特別漂亮——監督訊號是免費的：參數化空間裡障礙是解析的 AABB+速度，把未來時刻的盒解析光柵化成柵格（盒-體素求交是純索引算術，閉式）就得到預報的 ground truth，不需要模擬器、不需要感測器、不需要人工標註。這延續了 URPlanner「一切解析可算」的血統：ED2 用參數空間免費驗證擴散軌跡的碰撞，我們用參數空間免費生成預報監督——同一個哲學，不同的層。

新穎性邊界（誠實）：佔據預報本身在自駕領域存在（occupancy flow）；「速度估計當輔助任務」在移動機器人存在（MDRLAT）。沒人做過的是：manipulator 動態避障 + 佔據預報輔助 + 預報器同時服務「訓練期潛狀態」與「部署期體素屏蔽的檢查對象」這個雙重角色。一個模組同時解洞①與洞④，是本設計最強的結構之美。novelty 需在動手前再做一輪窮盡式檢索確認

System Design

系統設計：五個模組，每個都有 URPlanner 的血統

① 觀測 Observation

雙通道：(a) URPlanner Eq.2 原封不動的本體感知向量 [q, p_T, p_G, Δp, Δo, D]；(b) 以末端執行器工作區為中心裁剪的 k 幀佔據柵格（建議 32³×k=3，先二值、消融試距離場版本）。柵格在訓練期由參數空間解析光柵化生成（免模擬器），部署期由 depth/nvblox 體素化自然給出——同一種資料型態，sim-to-real 介面乾淨。

② 編碼器 + 預報頭（拱心石）

輕量 3D-CNN（或因子化 2D+1D 卷積省算力）→ 潛向量 z_t 進 policy/critic；同一主幹分出預報頭預測 t+Δt 佔據柵格，以解析光柵化的未來真值算 BCE/focal 輔助損失。λ_aux 權重消融。

③ 獎勵 D-UOAR-CT

r = r_pose + ζ_c·r_UOAR(Eq.10 不變) + ζ_s·r_swept（[t,t+Δt] 掃掠重疊積分）+ ζ_t·r_TTC（連續首穿時刻 τ* 的倒數懲罰）。全部由特權解析狀態算，只在訓練期存在（privileged learning / learning-by-cheating 的標準正當性）。關鍵框架修正見形式核心：這不是 reward shaping，是把離散碰撞代價「修正」為它的連續時間真值。

④ 演算法 APE2 + Shield

APE2 全保留（M×N+1 候選 + V_HPE 混合評估 + η 退火）。新增：對每個候選動作解區間首穿根 τ*(a)，剔除 τ*<Δt 的候選再取 argmax——與獎勵共用同一套根運算，額外成本近零。訓練期用物件級盒；部署期同一數學逐體素跑在預報佔據上（洞④的修法）。全部候選都不安全時的退避：解析縮步（縮放 Δq 使 τ*≥Δt——τ* 對步長單調，二分即可）。

⑤ 課程 + ED2

課程：靜態 → 慢速(0.05–0.2 m/s) → 快速(0.2–1.0) → 對抗攔截，成功率門檻自動晉級。ED2 保留並條件化於障礙運動模式；生成軌跡仍以參數空間連續時間檢查過濾（比原版離散過濾更嚴——順手的升級）。

整條管線維持 URPlanner 的三大不變量：平台無關（動作仍是關節增量、免 IK）、免模擬器訓練（狀態、獎勵、碰撞、柵格、預報真值全部解析可算）、最小距離無關（一切基於重疊與首穿時刻，不算 MD）。這是對「很多精神應該能從 URPlanner 讀到」的直接回應——不是借它的名字,是延用它的全部生成原理。

Formal Core

形式核心：一個重要的概念修正 + 問題定義

概念修正：D-UOAR-CT 不是 reward shaping,是目標修正（objective correction）

前幾輪一直把 swept 項當「塑形」討論、擔心 PBRS 政策不變性。本輪釐清：這個擔心用錯了地方。離散時間的 UOAR 是連續碰撞代價的有偏估計——障礙夠快時，兩個採樣時刻之間的真實穿透被系統性漏計（穿隧）。加入 swept 積分項不是在「不改變最優策略的前提下加速學習」（PBRS 的目標），而是把目標函數修正為我們真正想優化的連續時間碰撞代價——我們「就是要」它改變最優策略,改向真正安全的那個。PBRS 政策不變性只適用於可選的趨勢加速項 F（若保留）。這個一句話的框架修正會讓方法章嚴謹一個檔次,審稿人看得出作者知道自己在做什麼。

# 連續時間避障代價(真值)與其離散估計: C_true = ∫₀^Δt Σⱼ Σ_g l_jg(τ) dτ # l_jg(τ): 時刻 τ 的重疊長,τ 線性化下分段有理、閉式可積 C_disc = Δt · Σⱼ Σ_g l_jg(0) # 原 UOAR 的隱含假設:區間內重疊恆等於左端點值 bias = C_true − C_disc → 障礙速度↑、Δt↑ 時系統性非零 ⇒ 穿隧 # D-UOAR-CT = 用 C_true 替換 C_disc,加上首穿時刻懲罰: r_CT = r_pose − ζ_s·C_true/ΣLⱼ − ζ_t·β/(τ*+ε)

問題定義（POMDP,兩行版）

完整狀態 s_t =（機器人構型 q_t、目標、全部障礙盒的位姿與速度）為馬可夫;agent 觀測 o_t =（Eq.2 向量,k 幀柵格）。獎勵與屏蔽定義在 s_t 上（特權）,policy/critic 定義在編碼後的 (Eq.2, z_t) 上。拱心石的輔助任務保證 z_t 逼近障礙運動的充分統計量——這是整個訓練成立的資訊論前提,也是論文要用消融證明的第一件事（H2）。

The Lemma

保守性引理：把三種誤差全部收進膨脹盒,換一個無穿隧保證

引理（非正式陳述,論文中形式化）

已更正 · M0 測試抓到 本節初稿的 ε_lin = ΣᵢRᵢδθᵢ²/8 不成立——它漏掉多關節同動的 Hessian 交叉項,M0 的實證測試以反例否證(實測偏差 2.36mm > 該式上界 1.99mm)。修正後的嚴謹形式(經 |∂²p/∂qᵢ∂qⱼ| ≤ 2Rmax(i,j) 推導,已寫入 code/godynur/panda.py 並通過 200 例隨機驗證):設膨脹盒在原安全偏移 a_o 之上再膨脹 ε_total = ε_lin + ε_acc + ε_vel,其中:
ε_lin = ¼(Σᵢ √Rᵢ·|δθᵢ|)²(弦近似誤差;動作剪裁 |δθᵢ|≤0.05 rad/步時 ≤ 2cm;此界實測約 30 倍保守,以精確 Hessian 範數收緊是已知改進路徑)
ε_acc = a_max·Δt²/2（障礙等速假設誤差;a_max=2 m/s²、Δt=50ms → 2.5mm;人手級 a_max=10 → 12.5mm）
ε_vel = ε_v·Δt（速度估計誤差;ε_v=0.1 m/s → 5mm）
則區間根檢查 τ*(a) ≥ Δt 蘊含真實系統在 [t,t+Δt] 內無碰撞——即屏蔽在有界模型誤差下可證明保守,不存在假陰性（漏放危險動作）。總膨脹預算 ≈ 1.5–2.5cm,與 URPlanner 原本 5cm 級的 a_o 同量級,可接受。

這個引理是論文的「定理形狀物」。對照組全都給不出:DRP 無任何保證（原文承認）;PACS 有保證但需要可達集傳播機制;我們的保證只花幾行代數。證明骨架:弦近似偏差有界（幾何）+ 等速偏差有界（泰勒餘項）+ 誤差在盒座標系可加 → 膨脹後的盒包含真實掃掠體 → 區間不相交蘊含真實不相交。碩士生可以完整寫出的證明難度,恰到好處。
