# GO-DynURPlanner — 面向动态机械臂运动规划的连续时间碰撞推理

[English](README.md) | 简体中文

进行中的研究项目——新加坡国立大学（NUS）硕士课题，在 URPlanner
（[arXiv:2505.20175](https://arxiv.org/abs/2505.20175)）基础上向动态（障碍物移动）
环境延伸。

## 摘要

面向机械臂的标准强化学习运动规划器，每个控制步只在起始和/或终止构型处检查一次碰撞。
本项目在 URPlanner 的参数化空间（解析正运动学、线段-AABB 重叠、控制区间内的闭式首次
接触时刻）上构建了一个**精确的连续时间碰撞检查器**，并用它直接审计这一离散化假设。
在汇总的动态场景实验中，标准的逐步离散检查**漏检约 19% 的真实碰撞**，且漏检率在
准静态区间最差而非最好——这与"慢速障碍物更容易被检测到"的直觉恰恰相反。基于同一套
连续时间机制，本项目还有两个延伸贡献：一个**认证式动作屏蔽**（APE2-Shield），
通过在执行前认证候选动作，将训练期成功率提升约 4 倍、真实碰撞率降低约一半，把残余碰撞
转化为可证明不可避免的事件；以及一个**诚实的负结果**——用同一套连续时间代价去塑形奖励，
在修正了明显的病态项（一个无界的惩罚项）之后依然没有帮助，这说明这套机制的价值在于
度量与认证，而不在于奖励设计。柵格观测 + 占据预测辅助任务的扩展仍在进行中（目前
种子数偏低，结果尚无定论，见 `research-log/09_*`）。项目已经与导师的真实
（TensorFlow）URPlanner 实现做过对账：正运动学数值上完全一致，若干接口假设已据此
修正——见 `code/ASSUMPTIONS.md`。

## 快速开始

```bash
cd code
python3 -m pytest tests/ -q
python3 demo.py
```

需要 `numpy` 和 `pytest`。`torch` 仅在跑占据预测和 TD3 实验时才需要
（`experiments/m2_forecast_feasibility.py`、`godynur/forecast.py`、
`godynur/td3.py`）——核心库与其测试套件不需要。

## 关键结果

| 发现 | 数值 |
|---|---|
| 汇总的离散（S=1）检查相对连续真值的漏检率 | 19.0% [16.4%, 21.9%]（Wilson 95% 置信区间） |
| 0.1 m/s 时的漏检率（最差情况——准静态区间） | 45.6% [34.3%, 57.3%] |
| 2.0 m/s 时的漏检率 | 13.9% |
| 漏检碰撞的严重度，最大重叠长度：中位数 / 最坏情况 | 1.4 cm / 25.1 cm |
| 单步耗时：精确连续审计 vs S=16 离散（仍漏检 0.8%） | 1.50 ms vs 1.92 ms |
| 占据预测 kill test | 在 0.5 / 1.0 / 2.0 m/s 均通过（优于持久性基线与速度盲对照组） |
| **APE2-Shield**（认证式动作选择）vs 无屏蔽，同等训练预算 | 成功率 0.675 vs 0.175，真实碰撞率 0.24 vs 0.585 |
| 用同一套连续时间代价塑形奖励（D-UOAR-CT）vs 朴素离散奖励 | **诚实的负结果**——即便修正了无界惩罚项的病态，仍落后于朴素奖励（0.47/0.40 vs 0.65/0.25） |
| 柵格观测 + 占据预测辅助任务，seed 解耦后重跑 | 每臂 n=2 时尚无定论；有迹象显示该辅助任务降低训练崩溃风险，而非提升峰值表现——目前列为假设，非结论 |

完整推导过程见 `research-log/07_execution_m0_m1.html` 至
`09_shield_verdict_and_advisor.html`，这些结果如何组织成投稿见
`paper/PAPER1_PLAN.md`。
