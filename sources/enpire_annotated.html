Abstract

摘要

把「真實世界機器人學習」重構為「coding agent 可管理的優化過程」
Abstract

Achieving dexterous robotic manipulation in the real world relies heavily on human supervision and algorithmic engineering, which is a central bottleneck in the pursuit of general physical intelligence. Although emerging coding agents can generate code to automate algorithm search, their successes remain largely confined to digital environments.

We conjecture that the missing abstraction to automate robotics research is a repeatable feedback loop for real-world policy improvement: reset the scene, execute a policy, verify the outcome, and refine the next iteration. To bridge this gap, we introduce ENPIRE, a harness framework for coding agents with four core modules: Environment (EN), Policy Improvement (PI), Rollout (R), and Evolution (E).

Powered by ENPIRE, frontier coding agents can autonomously develop a policy to achieve a 99% success rate on challenging dexterous tasks (PushT, pin insertion, GPU insertion, zip-tie cutting). We propose two metrics — Mean Robot Utilization (MRU) and Mean Token Utilization (MTU) — to measure multi-agent physical autoresearch efficiency.

① 功能
四步推進：點出瓶頸（人類監督）→ 指出 coding agent 只在數位世界成功 → 提出缺的抽象（可重複的真實世界回饋迴圈）→ 落成系統（ENPIRE 四模組）+ 提出新指標。
② 邏輯角色
總命題節點。與 URPlanner 不同，本文的 thesis 不是「更好的演算法」，而是「一個讓 AI agent 能自主迭代真實機器人策略的框架抽象」——層次更高（meta-level）。
③ 技巧/漏洞
技巧：用「missing abstraction」把貢獻定位成「發現了一個缺失的範式」，比「我們做了個系統」格局更大。99% 成功率是強鉤子。漏洞：「autonomously」需打折——EN 階段仍需人類回饋（一次性成本），摘要用 amortize 邏輯辯護，但「自主」程度的邊界要到 §2.1 才清楚。
注意本文與 URPlanner 的抽象層級差異：URPlanner 改進「機器人怎麼規劃」，ENPIRE 改進「誰來改進機器人策略」（用 AI agent 取代人類研究員）。兩者可正交組合。
Section 1

引言：把人類從「保姆式」策略改進中解放

1 ¶1–3

Frontier policy training methods still rely on humans behind the scenes to participate in the data collection, evaluation with reset, and algorithm adjustment life cycle. Humans labor of babysitting policy improvement inevitably become one of the limiting factors of how quickly robots acquire dexterity.

Recent advances in autonomous research (autoresearch), powered by coding agents, show a promising path. However, unique challenges arise when extended to real-world policy learning. First, coding agents lack real-world environment interfaces for closed-loop hypothesis testing. Second, when scaling up autoresearch throughput across a robot fleet, it remains open how to select and verify hypotheses while maintaining efficiency.

ENPIRE decomposes skill acquisition into two autoresearch procedures: (i) EN — human-guided construction of an autonomous environment interface (one-time, amortized); (ii) PIRE — fully autonomous refinement from real-world feedback.

段落功能：問題定義 + 兩大挑戰 + 兩階段分解
① 功能
把「人類保姆」設為要消滅的瓶頸，列出兩個阻礙（缺真實環境介面、機群擴展時的假設選擇/驗證），對應兩階段設計（EN / PIRE）。
② 邏輯角色
「缺陷清單 → 模組對應」結構（與 URPlanner 同款修辭）。兩階段的切分是全文最關鍵的設計決策：把「需要人」與「完全自主」在時間軸上隔離。
③ 技巧/漏洞
技巧：「babysitting」一詞生動地把學術問題情緒化，提高記憶點。用「一次性成本 + 攤銷」化解「仍需人類」的質疑。漏洞：EN 的人類投入是否真能「攤銷」取決於任務間環境介面的可重用程度，論文以少數任務佐證，外推到任意任務的證據有限。
Section 2.1

階段一：從人類回饋建構環境 (EN)

把「物理互動」抽象成 coding agent 友善的結構化環境：安全約束 + 自動驗證 + 自動重置
2.1 三要素

Hard safety constraints: restrict the config space and kinematic behaviors to a safe operational limit. Violating limits triggers immediate task failure and automated reset, serving as a safeguard and a source of episode termination.

Automated verification: coding agents synthesize a binary reward function from procedural tool calls to distinguish outcomes. Given only a few minutes of success/failure demos, agents maximize prediction accuracy while minimizing latency (e.g., pin insertion reward from visual alignment + end-effector height + force estimates; latency <150ms).

Automated reset: tool calls (inspired by CaP-X) reset to the onset of the most challenging phase, strategically focusing learning on precision bottlenecks. Safety + verification + reset = the Environment Module (EN); with policies receiving visual-proprioceptive inputs = the Rollout module (R), exposed via immutable Gym APIs.

段落功能：環境模組的三支柱（方法論核心）
① 功能
定義讓 agent 能自主跑實驗的三個前提：安全約束（不撞壞硬體）、自動驗證（自己造獎勵函數判定成敗）、自動重置（無人快速重來）。
② 邏輯角色
EN 是整個自主迴圈能「無人運轉」的物理前提。其中「coding agent 自己合成獎勵函數」最具新意——把獎勵設計這個傳統上最需人類巧思的環節也自動化了。
③ 技巧/漏洞
技巧：用「重置到最難階段的起點」聚焦學習於瓶頸，是樣本效率的巧思。用 immutable Gym API 把環境凍結成穩定介面，讓 PIRE 階段可放心改演算法。漏洞：自動合成的二元獎勵準確率與延遲是品質天花板——若驗證函數有偏差，後續所有自主優化都在錯誤訊號上爬坡，論文未深究 reward hacking 風險。
自動驗證 = 自動獎勵函數。若 agent 合成的 reward 有系統性偏差，PIRE 會把策略優化到「看起來成功」而非「真的成功」（reward hacking）。雙相機 AND 邏輯（Fig.4）是部分緩解但非根治。
Section 2.2

階段二：從真實回饋自主改進策略 (PI + E)

coding agent 讀文獻、提假設、改訓練碼；多 agent 去中心化並行加速
2.2 PI 迴圈

The agent receives the task description and is granted write permissions to a streamlined training codebase. Through the PI module it reviews the literature, generates insights, formulates hypotheses, and directly modifies the training code (BC, RL algorithms) to optimize performance based on real-world automatic verification. To gather evidence, it invokes APIs to log trajectories, videos, and reward signals.

Multi-agent scaling (Evolution): deploy N agents across N robots to test N hypotheses asynchronously. Each branches off the same baseline codebase, collaborating via Git; agents spontaneously cherry-pick, copy, or merge successful recipes. Scaling parallel agent-robot pairs drastically reduces wall-clock time to discover a high-success recipe.

段落功能：兌現「自主研究」總命題
① 功能
描述 agent 如何像人類研究員一樣跑「讀文獻→提假設→改碼→看結果」迴圈，並用 Git 作為多 agent 協作的演化基質。
② 邏輯角色
論證的核心機制段。把「自主改進」具體化為「對訓練碼的可寫權限 + 文獻驅動的假設」。Evolution 用 Git merge 做假設選擇，是把軟體工程的協作範式移植到機器人研究的關鍵類比。
③ 技巧/漏洞
技巧：用「agents 自發 cherry-pick/merge」把多 agent 協作講得像開源社群，生動且可擴展。漏洞：「直接改訓練碼」意味著 agent 可能引入難以察覺的 bug 或不安全更新；論文未談如何防止演化過程退化或收斂到局部最優的訓練配方。
Section 3.1–3.2

實驗：啟發式學習 + 梯度式學習

三個前沿 coding agent（Codex/Claude Code/Kimi）× 真實 + 模擬任務
3.1 / 3.2

Tasks requiring precise reactive control: Push-T, Pin insertion (4mm holes), GPU-insertion, Zip-tie cutting. Benchmark three agents: Codex (GPT-5.5 xhigh), Claude Code (Opus 4.7 High), Kimi Code (Kimi K2.6 thinking).

On Push-T heuristic learning: all agents solve it in simulation; but the real world causes two of three to fail. Simulators offer consistent low-variance physics; real-world conditions are non-deterministic and time-varying (dynamics, friction, object movements). To improve real-world robustness, we encourage agents to explore and combine diverse learning methods (heuristics + gradient-based).

Pin insertion (gradient-based): agents achieve 50 consecutive successes by tuning BC, iterative BC + online rollout aggregation, offline/online RL with a BC-regularization term, and hyperparameters.

段落功能：能力驗證 + 揭示「真實世界才是難點」
① 功能
證明 ENPIRE 能驅動多種 coding agent 在真實精密任務上自主收斂；並用「模擬全過、真實兩敗」凸顯 sim-to-real 差距正是本框架的價值所在。
② 邏輯角色
實證支柱兼反差論證。「模擬簡單、真實才難」這個對比把全文價值錨定在「真實世界的自主研究」，與 §5 相關工作中「別人只在數位/模擬迴圈」形成主要區隔。
③ 技巧/漏洞
技巧：用「2/3 agent 在真實世界失敗」這種反直覺數據，反而強化「真實世界 autoresearch 是未解難題」的論點。漏洞：N=3 個 agent、少數任務，樣本量小；「鼓勵 agent 結合多種方法」帶有人類引導成分，與「完全自主」略有張力。
Section 3.3–3.5

機群擴展、經驗遷移與 VLA 協同

3.3 / 3.4 / 3.5

Fleet scaling: sizes 1/4/8 robots. In Push-T, scaling 1→8 agents cuts time-to-1.0-score from ~5h to ~2h; in pin insertion, from >1.5h to ~40min. Multi-agent also applies code-based domain randomization during reset (broader spatial variation than prior work → stronger robustness).

Continue learning: insights from multi-agent autoresearch transfer to similar novel tasks — appending documented training-recipe evolution to a new task's instructions (GPU insertion) yields high success.

Synergy with VLAs: ENPIRE integrates vision-language-action models (GR00T) with procedural tool calls for long-horizon manipulation — agent learns to hover over scissors, grasp, and cut a zip tie.

段落功能：證明「可擴展性」這一第二論點
① 功能
用機群規模 vs 時間的數據證明 ENPIRE 能用機器人資源換 wall-clock 時間；並展示經驗可跨任務遷移、可與 VLA 大模型協同。
② 邏輯角色
呼應摘要「scales on a robot fleet」的承諾。擴展性是把 ENPIRE 從「單機 demo」抬升為「可規模化研究基礎設施」的關鍵論點。
③ 技巧/漏洞
技巧：時間數字（5h→2h、1.5h→40min）直觀有力。VLA 協同把框架接到當紅大模型，擴大影響面。漏洞：擴展只測到 8 台，且 §3.6 自承「token 成本超線性增長」——擴展的經濟性拐點其實已隱現，但此處只報喜不報憂。
Section 3.6

量化 agent 資源利用率：MRU 與 MTU

3.6 新指標

Mean Robot Utilization (MRU) = fraction of research wall-clock time the robot is actively executing. Mean Token Utilization (MTU) = tokens consumed per minute. We further measure Tokens-to-Success and Time-to-Success. A perfectly resource-saturated agent pushes both toward 1; in practice neither reaches it for any frontier agent.

段落功能：提出新評估維度（次級貢獻）
① 功能
為「物理 autoresearch」這個新問題提出專屬效率指標：機器人利用率（稀缺物理資源）與 token 利用率（運算成本）。
② 邏輯角色
把貢獻從「一個系統」擴展為「一個可被測量、可被比較的研究範式」。指標化是讓後續研究能接續的制度性貢獻。
③ 技巧/漏洞
技巧：類比 GPU 利用率，把「機器人是新瓶頸資源」這個洞見指標化，立論點新穎。漏洞：MRU/MTU 仍是描述性指標，未給出「如何優化它們」的方法；且兩指標可能彼此衝突（多 agent 提高 GPU 用量卻降低單機 MRU）。
Section 4

限制：資源低利用與 token 成本超線性

4 限制

Underutilized resources: agents don't fully use robots while reading logs, writing code, debugging, or waiting for the LM backbone. As we scale robots, MRU decreases while GPU utilization increases — agent teams spend more time summarizing peer branches and less operating the robot.

Token cost grows super-linearly: as fleet size increases, token usage grows faster than the ideal linear trend. Increasing fleet size trades token efficiency for faster policy improvement — larger fleets reach success sooner but require a disproportionately higher token budget.

段落功能：誠實揭示框架的經濟學代價
① 功能
承認兩個核心代價：機器人在 agent「思考/協調」時閒置；機群越大 token 成本越超線性膨脹。
② 邏輯角色
論證的自我設限。把擴展性論點（§3.3）的反面誠實補齊，形成「速度 vs 成本」的權衡敘事，提升整體可信度。
③ 技巧/漏洞
技巧：用「trades efficiency for speed」把限制框成可選的權衡而非缺陷。漏洞：限制聚焦於「效率/成本」，迴避了更根本的問題——自動驗證的正確性、演化過程的安全性、long-horizon 任務的泛化天花板，這些更難的限制未被列入。
Section 5

相關工作：在硬體上跑迴圈才是關鍵差異

5.1–5.3

Code-as-Policies, ProgPrompt established executable code as the agent's action. Self-improvement loops need each trial cheap enough to repeat at scale — Minecraft skill libraries, Eureka reward design (Isaac Gym rollouts free). In every prior case, the loop closes on a cheap substrate (simulation/code); real-robot execution serves only as a sim-to-real or data target, never as the medium of iteration.

Two gaps: (1) no system autonomously runs and optimizes a physical robotics loop under an explicit hardware budget (classical robot scientists fixed the apparatus, never wrote their own tools); (2) no benchmark measures utilization of a scarce physical resource rather than capability or cost-per-paper. Our system addresses both.

段落功能：精準定位本文的獨特性
① 功能
用「迭代發生在哪種基質上」這條軸線重新組織所有相關工作，論證前人都在便宜基質（模擬/程式）上閉環，唯獨本文在真實硬體上閉環。
② 邏輯角色
全文差異化論證的最強段。把貢獻濃縮成兩個前人未填的 gap（硬體預算下的物理迴圈、稀缺物理資源的利用率指標），定位極其鋒利。
③ 技巧/漏洞
技巧：「the medium of iteration」這個切角極漂亮——不比演算法強弱，而比「迴圈跑在哪」，一舉把自己與整片文獻區隔。漏洞：這個區隔成立的代價就是貴（§4 已自承）；批評者可問：既然模擬便宜，為何不先在模擬把迴圈跑到極致再 sim-to-real？本文的回答（真實物理不可預測）說服力依賴 §3 那「2/3 失敗」的小樣本。
本文的立論軸 = 「迭代的媒介」。這與 URPlanner 的立論軸（規劃的計算成本）完全不同——交叉研究時，這正是兩篇可互補之處：URPlanner 提供便宜的真實-等價模擬基質，可能緩解 ENPIRE 的成本痛點。

作者核心主張（一句話版）：機器人靈巧度進步的真正瓶頸是「人類研究員的勞動」，只要為 coding agent 補上一個能在真實硬體上自動重置、自動驗證、自動迭代的回饋迴圈抽象（ENPIRE），AI agent 就能像研究員一樣自主把真實世界策略爬升到接近滿分，並能用機群規模換取速度。

◎ 論證最強處

相關工作的「迭代媒介」定位（§5）+ 99% 真實成功率。作者不在「演算法誰更強」上糾纏，而是提出一條全新比較軸——閉環迴圈跑在便宜基質還是真實硬體上——使前人工作整齊地落在「另一邊」，貢獻的獨特性無可爭辯；再以真實世界 99%／50 連續成功的硬數據兜底。立論角度與實證強度兼具。

△ 論證最弱處

「自主」程度與自動驗證的可靠性。「fully autonomous」其實建立在 EN 階段的人類回饋與人類「鼓勵結合多方法」之上；更關鍵的是整個自主迴圈爬坡的訊號來自 agent 自己合成的二元獎勵函數——若該函數有偏差，PIRE 會優化出「看似成功」的策略（reward hacking）。加上真實任務樣本量小（3 agent、少數任務、機群僅到 8 台），普適性證據偏薄。
