Part ④ · The Real Foundation

立身之本：不是「動態」，是 URPlanner 的解析可算性

深度思考後的判斷：URPlanner 真正獨一無二的，不是 UOAR 這個獎勵，而是它賴以成立的參數化解析空間——障礙=AABB、連桿=線段，使得狀態、獎勵、碰撞全部有封閉形式、不需要模擬器。「動態環境」是任務（大家都做），但能讓我們站得住的，是把這個解析性延伸到時間軸所獨家解鎖的能力。

立身之本（一句話） 因為 URPlanner 的幾何是解析的，它是少數能在不依賴模擬器、不計算最小距離的前提下，對一個控制區間 [t, t+Δt] 內「移動連桿 vs 移動障礙」做精確連續時間碰撞推理的框架。由此可解鎖三件別人做不到的事：(i) 精確的 swept-volume 重疊作為前瞻稠密獎勵；(ii) 連續 TTC（接觸前即有梯度）；(iii) 對 APE2 候選池的精確區間碰撞屏蔽（不會「穿隧」）。

為什麼這比「趨勢獎勵」更能立身：

  • 真正以 URPlanner 為基底：它是 URPlanner 定義式 (Eq.6–10) 往時間的直接、原理性推廣，而非外掛一套通用動態 RL 技巧。
  • 與競品錯開：EAAI 用 LSTM 學趨勢 + 距離獎勵；我們用解析 swept-volume 重疊 + 精確 CCD——機制不同，可辯護。
  • 可檢驗的強主張：「無模擬器即達精確連續碰撞無穿隧」是具體、可實驗證明的性質（離散步法面對快速障礙會穿隧——我們可重現此失敗並擊敗它）。
  • 查證背書：swept volume 目前只被用於碰撞偵測（NeuralSVCD 等），從未當 DRL 獎勵 2509.00499——空位真實存在。
Part ⑤ · Derivation

數學推導：把 UOAR (Eq.10) 推廣到連續時間

起點（URPlanner 靜態）。連桿 j 為線段、端點 p_s, p_e；對障礙盒 g，逐軸解 λ_near, λ_far（Eq.7），相交條件 Eq.8，重疊長 l = |λ_far−λ_near|·|p_s p_e|（Eq.9），UOAR 為歸一化總重疊（Eq.10）。一切在單一時刻、靜態幾何上計算。

步驟 1 — 把兩個運動都參數化到區間 [0, Δt]

  • 連桿掃掠：動作 a_t=[Δq_i] 令關節線性插值 q(τ)=q_t+(τ/Δt)·a_t；經正運動學得端點軌跡。小 Δt 下線性近似 p_s(τ)≈p_s(0)+u_s·τ，u_s=(p_s(Δt)−p_s(0))/Δt（兩端 FK 即得，解析、便宜）。
  • 障礙運動：盒中心 c_g(τ)=c_g(0)+v_g·τ（估計速度，區間內常值）。
  • 取相對運動（在盒 g 座標系）：p̃(τ)=p(τ)−c_g(τ) 仍對 τ 線性。於是 Eq.7 的 λ_near/λ_far 成為 τ 的有理函數，重疊長 l_jg(τ) 為分段平滑函數。

步驟 2 — 三個解析項（皆最小距離無關）

// (A) Swept 重疊：時間積分的前瞻懲罰 r_swept = −(1/ΣⱼLⱼ) · Σⱼ Σ_g ∫₀^Δt w(τ)·l_jg(τ) dτ (A)

τ 線性 ⇒ 被積式分段有理 ⇒ 每段閉式可積（或數點 Gauss 求積）。w(τ) 為近未來加權（如 e^{−κτ}）。當 Δt→0 退化回原 UOAR——嚴格相容。

// (B) 連續 TTC：接觸前即有梯度，補 UOAR「盒外零梯度」 τ* = min{ τ∈[0,Δt] : max(λ_near(τ)) = min(λ_far(τ)) } r_TTC = −β / (τ* + ε) (B)

τ* 為移動線段首次穿入移動盒的時刻，解低次有理方程的根即得。即使當前重疊=0（線段在盒外）τ* 仍有限 ⇒ 梯度非零——精確解掉反應式 UOAR 的致命傷，且不需最小距離。

// (C) PBRS 趨勢項：保證不改變最優策略（Devlin–Kudenko 時變版） Φ(s,t) = α·|r_swept(s,t)| F = γ·Φ(s',t+1) − Φ(s,t) (C)

逼近趨險 Φ↑ ⇒ F<0；遠離 Φ↓ ⇒ F>0——這就是你最初回憶的「看下一刻趨勢」。Devlin–Kudenko (2012) 已證時變勢能下政策不變性仍成立 White Rose，故此項只加速學習、不扭曲最優解。

// 完整 D-UOAR（連續時間版） r = r_pose + ζ_s·r_swept + ζ_t·r_TTC + ζ_F·F (D-UOAR-CT)

步驟 3 — 精確區間屏蔽（復用 APE2 候選池）

APE2 每步已生成 M×N+1 候選動作。每個候選定義一段連桿掃掠；以同一套 τ* 根運算判定區間內是否必碰，剔除 τ*<Δt 的候選，再於存活者取 argmax V_HPE：

ā_t = argmax_{a∈A_t, τ*(a)≥Δt} V_HPE(s_t, a) (Shield)

這是精確的區間碰撞無關性，非離散步抽樣——故不會讓快速細障礙「穿隧」而過（離散碰撞檢查的已知失效模式）。別人需另解 QP 合成安全動作 iKinQP-RL；我們零額外求解器，因 APE2 已備候選、URPlanner 已備解析幾何。

三項貢獻的共同根 (A)(B)(C)(Shield) 全部源於同一個解析操作——對 τ 線性化後解 λ(τ) 的根與積分。所以新增計算量 ≈ 「在 UOAR 之上多一遍區間根/積分」，與 URPlanner「便宜、可解析、平台無關」的精神一致；不需模擬器、不需最小距離、不需學一個趨勢網路。
Part ⑥ · Direction Verdict

裁決：主貢獻、第一篇核心。強主張＝「無模擬器即達精確連續無穿隧避障」。

Part ⑦ · Re-grounded Pitch

重構後的三句話定位

基底（確立）

URPlanner 的解析參數空間

狀態/獎勵/碰撞皆封閉形式、無需模擬器、最小距離無關、平台無關、免 IK。這是一切的根。

核心貢獻

解析性 × 時間 = 連續碰撞推理

swept 重疊前瞻獎勵 + 連續 TTC + 精確區間屏蔽；無穿隧、無模擬器、與趨勢競品機制錯開。

靈感（非基礎）

ENPIRE 式自動調參

選配的未來章節：在便宜參數空間自動搜尋獎勵權重。加速器，不是立論根據。

定稿標題建議 第一篇（核心、低風險）：「Continuous-Time, Minimum-Distance-Independent Collision Reasoning for Dynamic Manipulator Motion Planning」——賣點是 swept-overlap 獎勵 + 精確 CCD 屏蔽 + 無穿隧保證，全部源於 URPlanner 解析空間。把趨勢/速度當基本款內含，避開與 EAAI 正面比拼「誰的趨勢學得好」，改打「誰能在不靠模擬器下精確處理連續時間」。

仍未決、需下一輪迴圈處理的問題（誠實列出）：①線性化 p(τ) 在大 Δt 或高速關節下的誤差界，需推導或用保守包絡；②估計障礙速度的噪聲如何傳進 τ* 的可靠性（或改用區間/集員估計做 robust τ*）；③EAAI 競品的全文仍被付費牆擋住，須設法取得以精確界定重疊邊界；④swept 積分的即時性（目標 100Hz）需實測。這些是第一階段（M1–4）要關掉的風險。
