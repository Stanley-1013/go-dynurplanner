Master's-Level Extension Proposal

動態環境下機械臂的
無碰撞運動規劃：從 URPlanner 到 DynURPlanner

本文件先對 URPlanner（DRL 運動規劃的通用範式）與 ENPIRE（真實世界 agentic 自主研究框架）做交叉分析，再以 URPlanner 第一作者 Fengkang Ying 提出的碩士題目「Collision-free Motion Planning for Robot Manipulator in Dynamic Environments Based on DRL」為目標，提出一個具體、可發表的延伸研究設計——核心在於狀態表徵、獎勵函數、演算法三者如何從「靜態」躍遷到「動態」。

Paper A · URPlanner
便宜的「怎麼規劃」

參數化空間 + UOAR（免最小距離稠密獎勵）+ APE2 + ED2。優勢：平台無關、計算快 ~90×、免 IK。限制：僅限靜態障礙。

Paper B · ENPIRE
自主的「誰來改進」

EN/PI/R/E 四模組讓 coding agent 在真實硬體上自動重置/驗證/迭代策略。優勢：自主設計獎勵與演算法。限制：真實迭代昂貴、token 超線性。

Extension · DynURPlanner
動態的「會動的牆」

把 UOAR 升級為含「趨勢/預測」的時空獎勵、狀態加入障礙運動、APE2 加時序記憶；並用 ENPIRE 式自主研究在便宜參數空間中自動調出該獎勵。

Part ①

交叉分析：兩篇論文是正交而互補的

兩篇論文乍看都談「機器人 + 學習」，但它們作用在完全不同的抽象層級。URPlanner 改進「機械臂該怎麼算出一條無碰撞軌跡」；ENPIRE 改進「由誰、用什麼迴圈去把一個機器人策略迭代到好」。前者是「物件層」的演算法貢獻，後者是「元層」的研究流程貢獻。正因正交，它們可以相乘。

互補性：彼此的限制正是對方的優勢

URPlanner 的痛

獎勵與超參靠人手調

  • UOAR 權重 ζ、安全偏移 ao 都是人工試出的甜蜜點
  • 擴到動態場景時，「該怎麼設計動態獎勵」沒有現成答案
  • 但它有一個便宜到 0.06s 生成、解析可算的參數空間
⊗
ENPIRE 的痛

真實硬體迭代太貴

  • 真實 rollout 慢、token 成本隨機群超線性增長
  • 但它最擅長自動合成/精修獎勵、自動改訓練碼、自動調超參
  • 缺一個能讓它「便宜地試錯上萬次」的基質

這個 ⊗ 就是延伸研究第三條貢獻線（§⑥）的種子：把 URPlanner 的參數化空間當作 ENPIRE 的便宜基質，讓 coding agent 在其中自動搜尋「動態 UOAR」的形式與權重——一方解決另一方的限制。

Part ②

缺口：URPlanner 為什麼一遇到「會動的障礙」就失效

延伸題目不是憑空發明，而是直接踩在 URPlanner 自己留下的三個裂縫上（§II Eq.2、§III-B Eq.10、§VIII-D）。把這三點講清楚，延伸研究的「必要性」就成立了。

Gap 1 · 狀態盲視 狀態向量根本不編碼障礙物。URPlanner 的 s_t = [q_t, p_T, p_G, Δp, Δo, D]（Eq.2）只含關節角、工具/目標位姿與誤差，沒有任何障礙物資訊。靜態場景中，策略可以把固定障礙的位置「背」進網路權重；但障礙一旦移動，狀態就不再是馬可夫的——同一個 s_t 在障礙的不同位置會對應完全不同的最佳動作。這使問題從 MDP 退化為 POMDP。
Gap 2 · 反應式、盒外無梯度 UOAR 只在「線段已侵入膨脹盒」時才有非零訊號。由 Eq.10，r_UOAR = −Σ重疊長度 / Σ連桿長；當連桿仍在盒外，重疊長度 = 0，獎勵 = 0、梯度 = 0。靜態時這沒問題（位姿獎勵負責遠距引導）；但面對逼近中的動態障礙，等到偵測為「已侵入」往往為時已晚——機械臂沒有提前讓路的誘因。這是純反應式避障的典型病。
Gap 3 · 作者親自留下的鉤子 URPlanner §VIII-D 與結論明說兩件事：(i) 對隨機/變動障礙，須把障礙資訊納入狀態，並提議用「固定解析度的 3D 佔據柵格」維持狀態維度固定；(ii)「未來工作探索動態環境下的移動障礙，並可把人建模為移動障礙以支援人機協作 (HRC)」。換言之，動態環境是作者官方授權的延伸方向，且已給出狀態表徵的雛形。
Part ③

研究問題

狀態表徵：如何在維持狀態維度固定的前提下，讓策略同時感知障礙的位置與運動趨勢（速度/方向），使動態規劃重新近似馬可夫？佔據柵格、物件中心向量、時序堆疊三者的取捨為何？
獎勵函數：如何把只在「已侵入」才有訊號的反應式 UOAR，擴展為含「趨勢/預測」項的時空獎勵，使機械臂能提前對逼近的障礙讓路，且仍維持最小距離無關與解析可算？（核心，見 §⑤）
演算法：APE2 的混合評估與多候選探索，在 POMDP + 動態安全約束下要如何強化？是否需要時序記憶（recurrent）與安全 RL（CMDP/shielding）來保證碰撞率上界？
跨論文綜合：能否用 ENPIRE 式 agentic autoresearch，在 URPlanner 的便宜參數空間中自動搜尋上述動態獎勵的形式與權重，從而同時解決「人手調獎勵」與「真實迭代貴」兩個痛點？
Part ④

方法設計：DynURPlanner

沿用 URPlanner 的參數化空間骨架（障礙 = AABB、連桿 = 線段+膨脹盒），但在三個地方做動態化改造。設計原則：盡量維持「解析可算、最小距離無關、平台無關」三大原有優勢，只在必要處增加時間維度。

A

狀態表徵：感知運動趨勢

對應 RQ1 / Gap 1

對 URPlanner 的 s_t 補上障礙的時空資訊，三條候選路線（將做消融比較）：

  • 物件中心（低維）：每障礙 [中心, 範圍, 估計線速度 v]。維度低、可直接餵 MLP；需偵測/追蹤。
  • 時序堆疊佔據柵格（免追蹤）：採作者建議的固定解析度 3D 佔據柵格，堆疊最近 k 幀 → 用 3D-CNN/ConvGRU 編出固定維度潛在向量，讓網路自行推斷速度。
  • 混合：柵格編碼障礙場 + 顯式 EE/目標向量。
B

獎勵：反應 → 預測（見 §⑤）

對應 RQ2 / Gap 2

把 UOAR 升級為 D-UOAR（Dynamic UOAR），加入兩個維持「最小距離無關」的趨勢項：

  • 預測侵入（前瞻 Δt）：用估計速度把障礙盒前推 Δt，計算未來重疊長度並提前懲罰。
  • 逼近速率/勢能差：對「重疊勢能」的時間變化做 potential-based shaping，逼近扣分、遠離給分。
C

演算法：APE2 + 記憶 + 安全

對應 RQ3 / Gap 1,2

在 APE2 上做兩處增補：

  • 時序記憶：actor/critic 改用 GRU 或對狀態做幀堆疊，補回 POMDP 損失的時間資訊。
  • 安全 RL：把「動態碰撞率」設為 CMDP 約束（Lagrangian），或加 shielding 在執行前否決會撞的候選動作——天然契合 APE2「先生多候選再選優」的結構。
  • 課程學習：靜態 → 慢速 → 快速/對抗式移動障礙，逐步加難。
D

資料：ED2 條件化於障礙運動

對應 RQ2,3

把 URPlanner 的擴散式資料生成 ED2 條件化於障礙運動模式，自動合成大量「動態場景軌跡」做資料補償（DC），降低高維狀態帶來的樣本需求；生成軌跡仍以參數空間的連續碰撞檢測過濾。

Part ⑤ · 核心創新

趨勢獎勵：不只看「現在離牆多近」，更看「下一刻是要撞還是要離開」

這正是你回憶中的概念（已查證） 你記得的「離牆近、以位置來說獎勵可能低，但還要看下一瞬間是趨向撞牆還是離開，這也要算進獎勵」——這在文獻上有明確且成熟的對應。查證後最貼近你描述的是 Lütjens 等人「Missing Velocity in Dynamic Obstacle Avoidance based on DRL」(arXiv:2112.12465)：他們明確主張「只有位置不足以避開移動障礙，必須把速度/趨勢納入狀態，否則 agent 無法預測未來碰撞風險、無法及時讓路」，並用 LSTM-TD3（時序網路）解決。而 URPlanner 其實已部分內建此思想（APE2 的 Q 值），只是沒做到動態。它對應三個彼此相關、皆有文獻支撐的機制 ↓

① 它在理論上叫「基於勢能的獎勵塑形」(Potential-Based Reward Shaping)

Ng, Harada & Russell (1999) 證明：給原獎勵加上一個「勢能差」項 不會改變最優策略，卻能大幅加速學習。把「離障礙的危險程度」定義成一個勢能 Φ(s)，則塑形獎勵正是相鄰兩狀態的勢能差——這個差恰好編碼了「在逼近還是在遠離」的趨勢：

F(s, s′) = γ · Φ(s′) − Φ(s)
當機械臂逼近障礙：Φ 上升 → F < 0（扣分）；當它遠離：Φ 下降 → F > 0（給分）。即使「當前位置」很靠近牆，只要「下一刻在離開」，塑形項就給正回饋——完全是你描述的行為。

② 它在工程上等價於「逼近速率 / Time-to-Collision」項

勢能差在連續時間下就是危險勢能的時間導數。具體到 DynURPlanner，可令重疊勢能 Φ = Σ 預測重疊長度，其變化率 dΦ/dt 即「逼近速率」。也可改用更直觀的 TTC（碰撞剩餘時間）。查證發現這已是成熟做法：Damanik 等人的 「TTC-force Reward Shaping」(AAMAS 2023) 指出「預期碰撞時間比『到障礙距離』更能表達碰撞風險，因為它同時納入了距離與相對速度」——這正是「不只看位置、要看趨勢」的精確數學化；且 TTC 越小越危險、即使尚未接觸也有梯度，直接補上 Gap 2 的「盒外無梯度」。D-UOAR 的完整形式：

r_DUOAR = −[ overlapnow + α · overlappred(t+Δt) ] / ΣLj　+　β · ( Φ(s) − γΦ(s′) )
第一項 = 原 UOAR 的當前重疊（反應）；第二項 overlappred = 把障礙前推 Δt 後的預測重疊（前瞻）；第三項 = 勢能差 / 逼近速率（趨勢）。三項皆最小距離無關、解析可算，保住 URPlanner 的原始優勢。

③ URPlanner 其實已經有一半：APE2 的 VHPE

你的記憶裡還有「要看之後的成功/失敗機會」——這正是 URPlanner 的 APE2 在做的事。它的混合評估 V_HPE = η·Q_LTR + (1−η)·R_IR（Eq.17）裡，Q 值（long-term return）本身就是「這個狀態-動作未來會不會成功」的估計。所以「看未來趨勢」在 URPlanner 中已體現在動作評估層；DynURPlanner 要做的是把同樣的「未來導向」下沉到獎勵訊號層（用勢能差/預測項），讓即時回饋 R_IR 在動態場景也具前瞻性——兩層一致，互相強化。

一句話總結 把「危險」從位置的函數 Φ(位置) 升級為位置與趨勢的函數，再對它取「下一刻 − 現在」的差（或前推 Δt 的預測），就得到一個會「獎勵離開、懲罰逼近」的稠密訊號。這既是你記得的那個想法，也是 DynURPlanner 補上 URPlanner 動態缺口的關鍵獎勵設計——而且理論上（PBRS）保證不破壞最優策略。
Part ⑤·5 · 已查證文獻

文獻定位：這個方向已有人做，但留了一個 URPlanner 形狀的洞

依你的提醒，我上網查了實際研究而非憑記憶下結論。結論是：「動態障礙 + DRL + 趨勢獎勵」確實是活躍的研究線（這對提案是好事——代表方向被同行認可、可比較的基線充足），但現有工作幾乎都仍依賴距離 / 最小距離 / 光達來建構狀態與獎勵——而這正是 URPlanner 刻意拋棄的東西。所以 DynURPlanner 的真正空位是：把「最小距離無關、解析可算、平台無關」的 UOAR 範式，與「速度/TTC/勢能差」的趨勢獎勵首次結合。

已查證的相關工作它證實了什麼與本提案的差異（= 空位）
Lütjens et al. — Missing Velocity in Dynamic Obstacle Avoidance (DRL), arXiv:2112.12465位置-only 狀態不足以避動態障礙；須納入速度/趨勢；用 LSTM-TD3 時序網路。直接證實 Gap 1 與 §④-C 的記憶模組。它針對導航/點質點，仍用相對距離。本提案落在多自由度機械臂 + 重疊長度（非距離）。
Damanik et al. — TTC-force Reward Shaping (AAMAS 2023)「TTC 比距離更能表達碰撞風險，因納入相對速度」——你回憶的概念之數學化；TTC 提供盒外梯度。多機器人導航。本提案把 TTC 接到線段-包圍盒重疊並以參數空間解析計算。
Han et al. — RVO-shaped rewards, arXiv:2203.10229用「速度障礙 (VO/RVO) 面積 + 預期碰撞時間」做獎勵 + 雙向 recurrent，證實預測式獎勵 + 時序網路可行。機器人對機器人互避。本提案為單臂對外生移動障礙，且強調最小距離無關。
MDPI Appl. Sci. 2025 — Robotic Arms DRL with Adaptive Reward；以及 proactive HRC (ScienceDirect 2024)機械臂 + 隨機移動圓柱 / 人體部位，自適應/前瞻獎勵，HRC 成功率 ~93%。證實「人=移動障礙」路線可行（呼應 URPlanner §VIII-D）。仍以距離/接近度為主訊號。本提案的 MD-無關 + ENPIRE 自動獎勵設計（§⑥）為其所無。
Ng, Harada & Russell (1999) — Policy Invariance / PBRSF = γΦ(s′)−Φ(s) 不改變最優策略。為「趨勢項是否會把策略帶歪」提供理論保證。理論基石，本提案直接採用以約束 D-UOAR 的趨勢項。
查證後的修正結論（不過度宣稱） 「趨勢/速度/TTC 獎勵」與「動態機械臂避障 DRL」都不是本提案首創，這點要誠實。本提案的可辯護新穎性收斂為三點：(1) 把這些趨勢獎勵嫁接到 URPlanner 的最小距離無關、解析可算的重疊長度範式（現有動態工作仍用距離/光達）；(2) 在固定維度時空狀態下保住平台無關；(3) 用 ENPIRE 式自主研究在便宜參數空間自動設計該動態獎勵——這第三點目前文獻上尚無人做，是最強的差異化賣點。
Part ⑥

與 ENPIRE 融合：讓 agent 自動「發明」這個動態獎勵

§⑤ 的 D-UOAR 有許多需要人手定的設計選擇：前瞻時長 Δt、權重 α/β/γ、勢能 Φ 的形式、課程的難度排程……這正是 ENPIRE 最擅長自動化的東西。而 ENPIRE 最大的痛是「真實迭代太貴」，URPlanner 的參數空間恰好提供「解析可算、0.06s 生成」的便宜基質。兩者相乘得到第三條貢獻線：

綜合假設 在 URPlanner 的參數化空間中執行 ENPIRE 式 agentic autoresearch：coding agent 把參數空間當成「免費 rollout 的環境模組 (EN)」，自動合成並精修 D-UOAR 的形式與權重、自動調 APE2 超參與課程，僅在少數驗證階段才動用真實 Franka。預期同時達成：(i) 免去人手調獎勵；(ii) 把 ENPIRE 的 token/機器人成本壓到可接受；(iii) 得到一個比人手設計更穩健的動態避障獎勵。

這條線把「碩士可做的紮實工程題（DynURPlanner）」升級為「有頂會賣點的綜合題」——它正面回應了題目敘述中「state representation、reward function design、algorithm optimization」三個關鍵詞，又巧妙地把兩篇來源論文縫在一起。可作為論文的進階章節或第二篇投稿。

Part ⑦

實驗計畫

項目設計
動態場景輸送帶式等速障礙、擺動障礙、隨機遊走障礙、錄製的人手臂軌跡（HRC 前導）；障礙速度分級（慢/中/快/對抗）。
平台模擬：參數空間 + CoppeliaSim/Isaac 對照（沿用 URPlanner 設定）。真實：7-DoF Franka Emika Panda + 動捕或視覺追蹤的移動障礙。
基線① 原 URPlanner（靜態策略 + 週期性重規劃）② 人工勢場 (APF) / 動態窗口法 ③ MPC ④ 反應式 DRL（無趨勢項）⑤ DynURPlanner 各消融。
消融狀態三選一（物件中心 / 柵格 / 混合）；獎勵逐項加（當前重疊 → +預測項 → +勢能差）；演算法（APE2 vs +GRU vs +安全約束）。
指標成功率、動態碰撞率、軌跡長度、運動中最小動態間距、重規劃延遲、對障礙速度的魯棒性曲線、樣本效率。
假設驗證H1：含趨勢項的 D-UOAR 在中/高速障礙下碰撞率顯著低於反應式。H2：時序狀態 + GRU 優於單幀。H3：ENPIRE 式自動搜尋的獎勵 ≥ 人手最佳調參，且真實成本大幅降低。
Part ⑧

預期貢獻與新穎性

Contribution 1

D-UOAR 動態避障獎勵

首個「最小距離無關 + 解析可算 + 含前瞻/趨勢項」的動態障礙避障稠密獎勵；以 PBRS 保證不破壞最優策略，補上 UOAR 盒外無梯度的反應式缺陷。

Contribution 2

POMDP 化的固定維度時空狀態

系統比較物件中心 / 時序佔據柵格 / 混合三種表徵，落實 URPlanner §VIII-D 的柵格雛形，使動態規劃重新近似馬可夫並維持平台無關。

Contribution 3

便宜基質上的 agentic 自主研究

把 URPlanner 參數空間與 ENPIRE 自主研究迴圈縫合，自動設計動態獎勵與超參——同時解決 A 的「人手調獎勵」與 B 的「真實迭代貴」，是兩篇的原創綜合。

Part ⑨

風險與可行性

高維柵格 → 樣本爆炸

3D 佔據柵格使狀態維度暴增、訓練變慢（URPlanner 自己也只說「未來優化」）。緩解：先用低維物件中心表徵打通流程，再升級柵格；用 ED2 動態資料補償提升樣本效率。

速度估計噪聲

預測項依賴障礙速度估計，視覺追蹤有噪聲。緩解：前瞻 Δt 設短；用勢能差（一階差分）作為對噪聲更穩健的退路；卡爾曼濾波平滑。

sim-to-real 動態落差

動態接觸與時序在真實世界更難遷移（ENPIRE 的 2/3 失敗教訓）。緩解：沿用 URPlanner 平台無關設計 + 域隨機化 + 閉環重規劃（URPlanner 已驗證可數毫秒重規劃）。

自動獎勵的 reward hacking

若用 ENPIRE 自動合成獎勵，可能優化出「看似避障」的退化解。緩解：用參數空間的真值碰撞檢測作為獨立驗證；趨勢項以 PBRS 限制不改最優策略。
